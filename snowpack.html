Before I answer this question, let's take a look at how web project is built today.

Steps for build system to build a frontend app

  1. Read a source file
  2. Transpile it into browser readable format (e.g. ES5)
  3. Read dependencies and repeat step 2
  4. Concatenate all transpiled files into one big file
  5. Send the one big file to the browser

Thoughts

Snowpack uses rollup as underlying tool for transpiling and bundling. It still builds optimized bundle for production.

But it can let you choose to use webpack for bundling production build. So still compatible to our existing tools.

Downside is the learning cost for rollup system. There are also rough edges. I've tried to port Dash to snowpack, but failed.

Even though my topic is about snowpack, I'm really talking about the future of web development.

Future of frontend development

JS modules + HTTP2 can eliminate the need for a bundler.
